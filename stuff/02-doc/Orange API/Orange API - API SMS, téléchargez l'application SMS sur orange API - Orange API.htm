documentation de l'API

télécharger la fiche produit

L'API SMS est un service web d'envoi et de réception de SMS.
Sachez toutefois que votre identifiant (API Access Key) devra toujours être passé en paramètre dans l’URL de votre requête.

Les requêtes utilisent indifféremment les méthodes GET ou POST en mode HTTP ou HTPPS.
Le service web expose une interface dans un style REST-RPC.

1 . méthode

a.sendSMS

La méthode sendSMS vous permet d'envoyer un SMS dans le monde entier à partir d'une simple requête HTTP.

Tutoriel utilisant la méthode sendSMS :

  • - les tutoriels des API d'Orange
requête

http://run.orangeapi.com/sms/sendSMS.xml?id=[votre API Access Key]&from=[numéro court d'émission]&to=[numéro de téléphone]&content=[votre message à envoyer]&long_text=[activer ou non le SMS long]&max_sms=[nombre maximum de "parties" d'un SMS long]&ack=[activer ou non l'accusé de réception]&content_encoding=[type d’encodage à utiliser]

Détails des paramètres disponible ici >

Paramètre de la requêteDescription
id
type : string
champ : obligatoire

API Access Key est votre identifiant personnel lors de l'usage d'une API. L'API Access Key est disponible dans la rubrique mon espace personnel
from
type : string
champ : obligatoire

Numéro d'émission du SMS. Vous devez utiliser le bon numéro dans le paramètre "from" suivant votre destination.
Par default cette valeur est fixée à 38100.

numéro court

couverture

fonctionnalités

world International (destinations) émission de SMS

- le numéro d’émission qui apparait sur le téléphone mobile du récepteur est un numéro court local (différent pour chaque pays)
to
type : string
champ : obligatoire

Numéro du mobile de destination au format international.
ex. : 3312345678 (France) ou 441234567890 (Royaume-Uni)
content
type : string
champ : optionnel

Contenu du SMS, limité à 160 caractères
long_text
type : boolean
champ : optionnel

Permet d'activer ou non la fonctionnalité "SMS long".

Valeurs :
- "false" : seuls les messages de moins de 160 caractères peuvent être envoyés.
- "true" : le message sera découpé en plusieurs SMS de 134 caractères chacuns qui seront ensuite concaténés (regroupés) sur le téléphone mobile de destination.

Valeur par défaut : "false" 

Veuillez noter qu'un SMS long est limité à 34170 caractères, soit 255 SMS à concaténer.
max_sms
type : integer
champ : optionnel

Permet de définir le nombre maximum de "parties" (SMS à concaténer) pouvant constituer un "SMS long".

Valeurs : entier strictement supérieur à 1.

Dans le cas où cette limite serait dépassée, le SMS ne sera pas envoyé.
ack
type : boolean
champ : optionnel

Permet d’activer ou non l’accusé de réception.

Valeurs : "true" (activé) ou "false" (désactivé)
Valeur par default : "false"

Une fois le SMS envoyé avec ce paramètre à "true", utilisez l'API get history pour récupérer les informations suivantes :
- Le code de statut du SMS <ack_status_code>
- Le message associé <ack_status_msg> :

200 - Message delivered to handset
201 - Waiting for notification
400 - Error delivering the message
500 - Internal error

- La date et l’heure de réception du SMS <ack_date>
content_encoding
type : string
champ : optionnel

Ce paramètre permet de spécifier l’encodage du contenu à utiliser lors de la requête à l’API sendSMS.

Valeurs :
"gsm7" (valeur par défaut du paramètre)
Encodage des caractères au format GSM 7-bit.
Cet encodage (par défaut de la norme GSM) est aussi celui utilisé par défaut par notre API SMS.

"ucs2"
Encodage des caractères au format UCS-2.
Cet encodage permet l’utilisation de nombreux alphabets tels que l’Arabe, le Japonais et bien d’autres alphabets, mais aussi les caractères chinois.

Valeur par default : "gsm7"


Notes concernant l’utilisation de l’UCS-2 :
- la valeur du paramètre « content » doit être encodée en UTF-8.
- la valeur du paramètre « content » ne doit pas excéder 60 caractères.
- la fonctionnalité « SMS long » ne fonctionne pas avec cet encodage.
- le paramètre « from » doit être défini à « world » pour le monde entier ou « 38100 » pour la France uniquement.
- l’utilisation de la méthode POST doit être privilégiée au GET (l’encodage de l’URL dépendant du client HTTP).
- Il est nécessaire de spécifier l’encodage UTF-8 dans le header de la requête en POST : Content-Type: text/html; charset=UTF-8

b. receiveSMS recevez par mail ou par URL un SMS sur votre application

La méthode receiveSMS permet à votre application de recevoir un SMS envoyé depuis un mobile dans le monde entier.

Tutoriels utilisant la méthode receiveSMS :

  • - les tutoriels des API d'Orange
Vous devez tout d'abord configurer la réception de SMS dans la rubrique mon espace peronnel :
  • Définir vos mots-clés* (jusqu'à 5), qui nous permettent de rediriger les SMS vers votre application
  • Votre méthode de réception : par email, ou sur une URL (script), ou les deux

*Les mots-clés des API SMS et MMS sont partagés. Un mot clé rajouté pour l'API SMS fonctionnera aussi avec l'API MMS et inversement.



format du SMS à saisir sur le mobile

Le SMS envoyé par le mobile doit être sous la forme :

  • votre mot-clé (exemple: orange)
  • un espace ou un saut de ligne
  • le contenu du SMS

Exemple : « orange salut, comment ça va ? »
Le SMS ne doit pas dépasser 160 caractères (les caractères suivant seront tronqués)



méthode de réception

Nous vous proposons deux méthodes de réception, à paramétrer dans mon espace personnel

par email
Une fois que vous avez enregistré votre adresse e-mail, la fonction receiveSMS acheminera le SMS directement dans votre boîte de réception sous cette forme :

De: apache@orange.fr
A: adresse email
Date: mer. 01/07/09 18:34
Objet: Sms from [numéro de mobile émetteur] to [numéro court de réception] on 2009-07-01 16:34:09.733?

[mot clé + contenu du message]


par URL
Une fois que vous avez enregistré l'URL de votre script (exemple: http://www.monsite.com/fichierReception.php), la fonction receiveSMS acheminera le SMS vers cette URL en utilisant une requête HTTP GET avec les paramètres suivants :

http://www.monsite.com/fichierReception.php?api=[méthode api utilisé]&from=[numéro de mobile émetteur]&to=[numéro court de réception]&content=[contenu du SMS reçu]

Nous avons créé un tutoriel qui vous donne un exemple de code pour envoyer automatiquement un SMS MT en réponse au SMS MO de l’utilisateur final.



2 . exemple de code

envoyer un SMS en quelques lignes de code !

  • 1. copiez/collez ce code dans un fichier PHP.
  • 2. remplissez les paramètres API Access Key, FROM, TO et CONTENT.
  • 3. chargez le fichier sur votre serveur Web.
  • 4. exécutez votre page.

<?php

// votre API Access Key
$access_key = "XXXXXXX";

// adresse de l'API sendSMS
$adresse = "http://run.orangeapi.com/sms/sendSMS.xml";
// shortcode d'émission : 20345 (Orange France), 38100 (Multi-opérateur France), 967482 (Orange UK), 447797805210 (international) ou world (international)
$from = "38100";
// numéro de téléphone du destinataire (au format international)
$to = "336XXXXXXXX";
// activation du SMS long
$long_text = "true";
// limite du nombre de SMS à concaténer
$max_sms = "3";
// activation de l'accusé de réception
$ack = "true";
// type d'encodage
$content_encoding = "gsm7";
// contenu du SMS
$content = urlencode("votre contenu ici !");

// exécution de la requête> envoi du SMS
$fd = file_get_contents($adresse . "?id=" . $access_key . "&from=" . $from . "&to=" . $to . "&content=" . $content . "&long_text=" . $long_text . "&max_sms=" . $max_sms . "&ack=" . $ack . "&content_encoding=" . $content_encoding);

// Affichage de la réponse de l'API
$xml=simplexml_load_string($fd);
echo($xml->status->status_code . " - " . $xml->status->status_msg);

?>
